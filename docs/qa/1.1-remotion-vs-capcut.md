### Sistema visual Alturense observado no CapCut

O vídeo `altura11.mp4` revela 4 componentes visuais **permanentes** que o template Remotion actual **não produz**:

1. **Moldura decorativa** — padrão de azulejos portugueses (azul/branco) ao longo do lado esquerdo + sprig de oliveira ao longo do lado direito. Presente em todos os frames.
2. **Header textual permanente** — 3 linhas centradas no topo:
   - "ESPECIAL DE SÁBADO" (título em maiúsculas, font serif)
   - "(à moda do Altura)" (subtítulo em itálico, font serif)
3. **Card carimbo central** — selo/autocolante sobre o prato com o nome da refeição em caixa alta, font sem serifas, alinhamento centrado (ex: "CARAPAUS FRITOS"). Estilo "stamp" — aparenta ser uma camada sobreposta fixa, não uma animação de entrada.
4. **Footer permanente** — faixa laranja sólida no rodapé: "Take-Away · Snack Bar Alturense · Altura" (com separadores · ou •).

Estes 4 componentes **definem a identidade visual do Alturense** e são o que torna o vídeo reconhecível como "daquele restaurante". Sem eles, o vídeo Remotion parece um reel genérico.

### Veredicto global AC-C2

**`FAIL`** por gap de requisitos.

Resumo técnico:
- 0 critérios em PASS absoluto (Música PASS técnico mas CONCERNS operacional; Legendas N/A)
- 3 critérios em CONCERNS (pacing, música, cores)
- 2 critérios em FAIL (overlays, tom geral)
- 1 critério N/A (legendas)

Interpretação: **a Story 1.1 omitiu o sistema visual Alturense**. Os AC escritos descrevem um reel genérico com paleta quente e overlays configuráveis — e é exactamente isso que o @dev entregou. Mas o workflow real do Alturense produz peças "cartaz digital" com 4 componentes visuais permanentes que a Story 1.1 não menciona. O template Remotion é **funcionalmente completo para o scope declarado**, mas **insuficiente para substituir o CapCut real**, que é a métrica de sucesso (R4 mitigation).

### Não é FAIL do @dev

@dev (Dex) implementou com rigor técnico o que a Story descreve:
- Q-ARQ-3 day-1 integral (5 interfaces + EditingProvider + 4 stubs)
- Composition 1080×1080 30fps com TransitionSeries + 3 cenas + 2 transições fade
- Overlays configuráveis via props (TextOverlay, PriceBadge)
- MusicLayer com fade-in/out
- BurnedSubtitle stub-safe (Q-ARQ-2)
- PT-PT integralmente cumprido
- Documentação inline extensa
- 3 decisões técnicas documentadas e justificadas

O FAIL é exclusivamente de *scope*, não de implementação. Código deve ser preservado — vai servir de base para a Story 1.2.
