## Story

**Como** Eurico Operator,
**quero** um template Remotion funcional para o cliente Alturense que produza vídeos Facebook 1:1 de 20-30s com qualidade editorial equivalente ou superior ao meu workflow manual CapCut,
**para que** a Camada 3 (edição programática) esteja validada como fundação do pipeline `daily-content-delivery` antes de arrancar o workflow completo.

## Contexto e Razão de Ser

Esta é a **primeira story do Passo C** (implementação) do projecto VIDEO-AI-PT.

**Mandato explícito do PRD v1.0 §11.1 e §13.5:**
> Template "Alturense-default" em Remotion → "Mitigação R4 — deve ser primeira story do Passo C (antes do workflow `daily-content-delivery`)"

**Riscos mitigados por esta story:**
- **R4** (ALTA/ALTA) — Camada 3 Remotion não atinge qualidade CapCut no primeiro mês
- **R20** (MÉDIA/ALTA) — learning curve Remotion atrasa v1.0 em 1-2 semanas
- **R9** (ALTA/ALTA se não mitigado) — direitos de música: biblioteca royalty-free obrigatória nesta story

**Scope desta story: TEMPLATE REMOTION (Camada 3) + contratos Provider (Q-ARQ-3).**
A integração com `script-writer` Reel (Camada 1 plan) é input assumido de sessões futuras — não é output desta story.

**AC-B2 — Composition principal**
`AlturenseComposition.tsx` deve:
- Aceitar props `AlturenseCompositionProps` conforme interface do scaffold (ver `docs/research/samples/remotion-alturense-scaffold.md` §Interface de dados)
- Composition: 1080×1080, 30fps, duração 20-30s configurável
- Estrutura: `TransitionSeries` com 3 sequências (Clip1Scene, Clip2Scene, FinalScene) ligadas por transição `fade` de 0.5s (15 frames a 30fps)
- `<Audio/>` para música royalty-free ao longo de toda a composition
- `<BurnedSubtitle/>` condicional (activo quando `subtitlesSrtPath` presente)
- Usar `@remotion/transitions` para fade — NÃO implementar transição manual

**AC-B3 — Overlays de texto configuráveis**

**AC-B4 — Legendas queimadas via `<BurnedSubtitle/>`**

**AC-B5 — Música royalty-free**

### Q-ARQ-3 — Abstraction Pattern Day-1

Esta é a decisão mais importante da arquitectura v1.0. **NUNCA chamar Remotion directamente** de uma task ou workflow — sempre via `EditingProvider.render({...})`.

Interface completa conforme PRD §8.3:
```javascript
// CommonJS, Node 18
EditingProvider {
  async render({composition_spec, assets, output_spec})
    → {output_path, cost_eur, latency_ms, render_logs}
}
```

`composition_spec` é o JSON com props do `AlturenseCompositionProps`. `assets` são os paths locais dos clips/designs. `output_spec` especifica o path de output e codec.

[Fonte: `PRD.md §8.3 Q-ARQ-3`]

### Remotion — Notas de Setup

- Remotion v4+ usa `@remotion/cli`, `@remotion/core`, `@remotion/player`
- Render local: `npx remotion render` usa headless Chromium incluído no pacote
- FFmpeg: instalado separadamente ou via `npx remotion install ffmpeg`
- TypeScript: Remotion é TS-first; squad usa CommonJS, mas o template Remotion pode ter `tsconfig.json` próprio (`.tsx` dentro da pasta do template)
- Licença Remotion v1.0: gratuito para ≤3 developers [KC-2026-01] — verificar em `https://www.remotion.dev/license` antes de adicionar mais operadores (R18)
